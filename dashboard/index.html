1. Latar belakang dan tujuan

Supply chain modern membutuhkan sinyal risiko sebelum masalah sampai ke pelanggan. Proyek ini dibuat untuk menjawab tiga kebutuhan utama: memahami keterlambatan pengiriman, melihat hubungan risiko dengan profitabilitas, dan membuat baseline model untuk memberi skor risiko order sebelum fulfillment selesai.

Dataset yang digunakan adalah DataCo Smart Supply Chain. Karena dataset bersifat publik dan historis, project ini diposisikan sebagai portfolio decision support prototype, bukan sistem produksi SLA.

2. Alur pembuatan project

Data preparationRaw CSV dibersihkan, nama kolom distandardisasi, fitur tanggal dibuat, dan output publik dipisahkan dari data mentah.
KPI martsPipeline membuat delivery, profitability, region shipping, category, customer market, dan monthly trend marts.
ModelingLogistic Regression dan Random Forest dibandingkan untuk prediksi late delivery risk tanpa post shipment leakage.
DashboardData mart dan model output dikompilasi ke JSON agar dashboard bisa berjalan statis di Vercel.

3. Isi dashboard

  • Overview: ringkasan order, sales, profit, late risk, model recall, distribusi delivery status, dan tren bulanan.
  • Delivery Risk: late rate by shipping mode dan kombinasi region x shipping mode yang paling berisiko.
  • Profitability: kategori, market, region, sales, margin, dan segmen high revenue namun low margin.
  • Customer Region: performa customer segment berdasarkan market untuk melihat segmentasi risiko dan profit.
  • Model Insights: feature importance, risk score sample, dan order dengan probability risk tinggi.
  • Recommendations: rekomendasi untuk risk queue, monitoring fulfillment, dan evaluasi profit quality.

4. Kenapa Random Forest dipilih

Random Forest dipilih bukan karena accuracy tertinggi. Logistic Regression memiliki accuracy 70,02%, sedangkan Random Forest 69,57%. Tetapi tujuan bisnisnya adalah menangkap order yang berisiko telat. Karena itu model dipilih berdasarkan composite score: 50% recall late class, 30% F1 late class, dan 20% ROC-AUC.

Random Forest menang tipis karena recall late class lebih tinggi. Dalam supply chain, missed risky shipment biasanya lebih mahal daripada over-flagging order aman untuk monitoring.

5. Kenapa accuracy 69,57% masuk akal

Model sengaja dibuat anti leakage. Field seperti actual shipping days, final delivery status, shipping gap, dan order status tidak dipakai karena baru diketahui setelah pengiriman terjadi. Jika field tersebut dipakai, accuracy bisa terlihat lebih tinggi, tetapi model tidak lagi realistis untuk prediksi sebelum fulfillment.

Accuracy juga dibatasi oleh karakter dataset publik. Faktor penting seperti cuaca, kapasitas gudang, kapasitas carrier, live inventory, jarak rute aktual, dan backlog harian tidak tersedia. Karena itu 69,57% dibaca sebagai baseline realistis, bukan angka final produksi.

6. Kesimpulan dan arah lanjutan

Project ini menunjukkan proses end to end dari data engineering, EDA, machine learning, dashboard design, sampai deployment. Nilai utamanya ada pada cara dashboard dan model menjawab masalah bisnis secara jujur tanpa target leakage.

Pengembangan berikutnya dapat mencakup threshold tuning, PR-AUC, cost-sensitive evaluation, calibration curve, LightGBM atau XGBoost, serta integrasi data operasional seperti carrier capacity, warehouse backlog, route distance, cuaca, dan live inventory.
